# サムライ経済学 台本マニュアル v2

**目的**: このマニュアルの通りに進めれば、誰でもサムライ経済学の台本が書ける  
**抽出元**: 16本の競合台本分析から共通パターンを抽象化  
**更新日**: 2026-02-15  

---

# ═══ 第一部：3つの原則 ═══

台本の全てはこの3つに集約される。迷ったらここに戻る。

### 原則1：数字を人間に変換する

マクロの数字をそのまま出さない。**1人の1日**に翻訳する。

```
✗「インフレ率130万%」
○「1ヶ月の給料で卵12個しか買えない」
```

**翻訳の3段階**:
① 数字そのもの（論理）→ ② 日常比較（比較）→ ③ 映像化（視覚）

### 原則2：専門用語を出したら同じ文で言い換える

視聴者が「？」になった瞬間に離脱する。

```
✗「アービトラージが発生し…」
○「安い国で買って高い国で売る業者、つまりアービトラージが…」
```

### 原則3：結論を押しつけず、問いを返す

「投資しろ」「逃げろ」ではなく「あなたはどう思いますか？」で終わる。

# ═══ 第二部：台本の共通骨格 ═══

**全てのシナリオ構造に共通する骨格**。どの型を選んでも、この流れは変わらない。

```
[フック] → [CQ提示] → [背景] → [核心の解説] → [クライマックス] → [教訓＋日本ブリッジ] → [CTA]
```

| パート | 役割 | 尺の目安 |
|--------|------|---------|
| **フック** | 手を止めさせる | 全体の5〜10% |
| **CQ提示** | 「最後まで見る理由」を渡す | 5% |
| **背景** | 理解の土台を作る | 10〜15% |
| **核心の解説** | CQに答え始める | 30〜40% |
| **クライマックス** | 感情の最高点 | 10〜15% |
| **教訓＋日本ブリッジ** | 「自分の話」に変換する | 10〜15% |
| **CTA** | コメント→登録→次回 | 5% |

# ═══ 第三部：4つのシナリオ構造 ═══

16本の台本を分析した結果、全ての台本は**4つの構造**のどれかに分類できる。  
同じテーマの続編を作る時は**必ず別の構造を選ぶ**。

---

## 構造A：物語型（人が主役）

**一言**: 1人の人間の体験を通して、国や経済の問題を描く

**使う場面**:
- その国に住む（住んでいた）当事者がいる
- 個人の体験が国の問題を象徴している
- 感情に訴えたい（初回・シリーズ1本目に最適）

**基本フレーム**:

```
① フック（衝撃 or 日常→衝撃）
② 語り手の登場（名前・年齢・職業）
③ 天国パート（かつての豊かさ・平穏）
④ 崩壊の始まり（最初の異変・伏線）
⑤ 崩壊のメカニズム（構造解説を挟む）
⑥ どん底（感情の最深部。五感で描く）
⑦ 転換 or 脱出（希望 or 決意）
⑧ 教訓＋日本ブリッジ＋CTA
```

**感情曲線の形**: 山型（上がって→落ちて→少し戻る）

```
    天国         
   ╱    ╲         教訓
  ╱      ╲       ╱    
 ╱        ╲     ╱     
╱          ╲   ╱      
フック       どん底         CTA
```

**バリエーション**:

| 変形 | 特徴 | 向いている場面 |
|------|------|--------------|
| **一人称体験** | 語り手＝当事者が「私は」で語る | 国家崩壊・脱出・生活崩壊 |
| **3世代クロニクル** | 祖父母→親→自分の3世代で歴史を描く | 人権・自由・長期的変化 |
| **サバイバルマニュアル** | 脱出・対策を具体的に手順化する | シリーズ完結編・実用性重視 |

**この型のポイント**:
- 語り手には**名前・年齢・職業**を必ず与える
- 天国パートで**五感**（匂い・音・手触り）を入れる → 崩壊の落差が最大化する
- 崩壊は**段階的に**描く（食→医療→死のように）
- 結末は冒頭と呼応させる（**リング構造**）

---

## 構造B：謎解き型（問いが主役）

**一言**: 「なぜ？」「誰が得した？」の問いを解き明かしていく

**使う場面**:
- 事件の「裏側」がある
- 「誰も言えないこと」がテーマ
- 視聴者が結末を知っている（原因が知りたい）
- シリーズ2本目に最適（1本目で物語型→2本目で謎解き型）

**基本フレーム**:

```
① フック（結果をいきなり叩きつける）
② CQ提示（「なぜ？」「誰が得した？」「次に何が起きる？」）
③ 事実整理（5W1Hで「何が起きたか」を先に潰す）
④ ゲート宣言（「ここまでが表。ここから裏」）
⑤ ナンバリング回答（「3つの理由」を1つずつ開封）
⑥ 歴史的前例（過去にも同じことがあった）
⑦ 勝者と敗者の整理（「誰が得して、誰が損したか」）
⑧ 教訓＋日本ブリッジ＋CTA
```

**感情曲線の形**: 階段型（好奇心が段階的に上がる）

```
                    回答③
              回答②╱     
        回答①╱   ╱ 歴史
  CQ   ╱   ╱   ╱  ╱╲
  ╱╲  ╱   ╱   ╱  ╱  ╲教訓
 ╱  ╲╱   ╱   ╱  ╱    ╲
╱   事実  ゲート        CTA
フック   整理
```

**バリエーション**:

| 変形 | 特徴 | 向いている場面 |
|------|------|--------------|
| **事件の裏側** | 「なぜ今？」を地政学的に分析 | 軍事行動・外交問題 |
| **タブー解禁** | 誰も口にできない問題をデータで語る | 移民・人種・社会分断 |
| **主張検証→否定** | 「◯◯は本当か？」→データで検証→半分嘘 | 通説の反証・後編向き |

**この型のポイント**:
- **ゲート宣言**を必ず入れる（「ここまでが表向き。ここから本質」）
- 回答は**3つにナンバリング**する（記憶に残り、引用される）
- タブーに触れる時は**予防線**を先に張る（「これは誰かを責める話ではない」）
- 「感情ではなく数字で語る」と宣言すると信頼が生まれる

## 構造C：連鎖型（時間が主役）

**一言**: 出来事がドミノ倒しのように連鎖していく過程を追う

**使う場面**:
- 1つの判断・事件が連鎖的に問題を拡大した
- 「現在進行形」の事件を追っている
- 時間軸で追うと因果関係が明確になる

**基本フレーム**:

```
① フック（最も衝撃的な結果 or 「今この瞬間」）
② CQ提示＋時系列の起点を宣言
③ ドミノ①（最初のトリガー）
④ ドミノ②（連鎖反応）
⑤ ドミノ③（さらなる拡大）
⑥ ドミノ④（クライマックス＝最も深刻な帰結）
⑦ 「その後」（3年後・現在どうなったか）
⑧ 教訓＋日本ブリッジ＋CTA
```

**感情曲線の形**: 右肩上がり（エスカレーション）

```
                    ドミノ④
              ドミノ③╱╲    
        ドミノ②╱   ╱  ╲ その後
  ドミノ①╱   ╱   ╱    ╲╱╲
     ╱  ╱   ╱   ╱        ╲
    ╱  ╱   ╱   ╱     教訓  CTA
フック               
```

**バリエーション**:

| 変形 | 特徴 | 向いている場面 |
|------|------|--------------|
| **時系列ドミノ** | 日付を追って連鎖を可視化 | 政策の連鎖的破壊 |
| **リアルタイム報道** | 「今この瞬間」の緊迫感 | 現在進行形の事件 |
| **告発エスカレーション** | 問題を6〜8層積み上げる | 「勝ち組」の裏側暴露 |

**この型のポイント**:
- **「わずか◯日後」「まだ終わりじゃない」**のリフレインで加速感を演出
- 各ドミノの間に**息継ぎ**（ブラックユーモア・バランスデータ）を入れる
- 連鎖の途中に**「ここでこう思うかもしれない」**と逃げ道を塞ぐ
- 「その後」パート（他チャンネルがやらない追跡）を入れると差別化できる

---

## 構造D：解説型（仕組みが主役）

**一言**: 経済の仕組みや構造を「なるほど」の連続で理解させる

**使う場面**:
- 日本国内の経済テーマ
- 仕組みの理解そのものが価値（金利・為替・株価）
- 事件より「構造」が面白い

**基本フレーム**:

```
① フック（日常の違和感 or 逆説）
② 語り手（佐藤さん等＝視聴者の分身）の登場
③ 問題の分解（「3つの要因がある」）
④ 要因①の解説＋日常例で翻訳
⑤ 要因②の解説＋シミュレーション
⑥ 要因③の解説＋歴史データ
⑦ 全体像の統合（「つまりこういう構造」）
⑧ 「あなたの家計では？」＋CTA
```

**感情曲線の形**: 階段型（「なるほど」が積み上がる）

```
              要因③
        要因②╱    統合
  要因①╱   ╱   ╱╲
     ╱   ╱   ╱    ╲
    ╱   ╱   ╱  自分事  CTA
フック                    
```

**バリエーション**:

| 変形 | 特徴 | 向いている場面 |
|------|------|--------------|
| **エッセイ型** | 視聴者の分身（佐藤さん）が疑問を追う | 日経平均・給料・物価 |
| **二重視点型** | 生活者と専門家の2つの視点を行き来 | 為替・金利・投資 |
| **シミュレーション型** | 具体的な数字でステップ計算する | 住宅ローン・家計・税金 |

**この型のポイント**:
- **語り手＝視聴者の分身**にする（「佐藤さん」のような普通の人）
- 仕組みは**3つ以内に分解**する（4つ以上は記憶できない）
- 必ず**日常の行為に翻訳**する（「電気をつけるたびに円が出ていく」）
- 結論は**「答えは人それぞれ」**でOK。押しつけない。

---

## 構造の選び方

```
テーマが決まったら、以下の2つを問う：

Q1. 何が一番面白いか？
  → 「1人の人生」が面白い → 構造A：物語型
  → 「裏側・真相」が面白い → 構造B：謎解き型
  → 「連鎖・加速」が面白い → 構造C：連鎖型
  → 「仕組み・構造」が面白い → 構造D：解説型

Q2. 続編か？
  → YES → 前回と違う構造を選ぶ
  → NO → Q1の答えをそのまま使う
```

# ═══ 第四部：導入の書き方 ═══

導入＝台本全体の10%。5000字の台本なら**400〜500字、8〜12文**。

## 3ステップで書く

```
Step 1：フック（1〜4文） → 手を止める
Step 2：語り手の登場（3〜4文） → 信頼を作る
Step 3：CQ＋約束（2〜3文） → 最後まで見る理由を渡す
```

---

### Step 1：フックの書き方

**フックの原則**: 1文目で「え？」と思わせ、4文以内でCQへの橋を架ける。

**フックには4つの戦略がある**（これを組み合わせる）:

| 戦略 | 心理効果 | やり方 |
|------|---------|--------|
| **衝撃の数字/事実で入る** | 情報ギャップが生まれる | 前置きゼロで最も驚く数字を出す |
| **日常から入って壊す** | 安心→衝撃の落差が大きい | 何気ない日常を3〜5文並べ、最後の1文で世界を壊す |
| **結果から入る** | 「なぜ？」が爆発する | 理由も経緯もなしに結果だけ叩きつける |
| **自分の体験で入る** | 即座に共感が生まれる | 語り手自身の体験（病院・買い物・食卓）で始める |

**フックの文字数**: 1文あたり**6〜20字**。短いほど衝撃が大きい。

---

### Step 2：語り手の登場

```
[名前] ＋ [年齢] ＋ [職業] ＋ [状況]（1〜2文）
→
「なぜこの話をするのか」の動機（1〜2文）
→
前回動画との接続（あれば、1文だけ）
```

**語り手の種類**:

| 種類 | いつ使うか |
|------|----------|
| **海外の当事者**（「私の名前は◯◯」） | 海外テーマで個人の体験を語る時 |
| **視聴者の分身**（「佐藤さん」的な普通の人） | 日本国内テーマ、仕組みの解説 |
| **解説者**（投資家・分析者の視点） | 俯瞰的に構造を語る時 |

---

### Step 3：CQ（セントラルクエスチョン）の設計

**CQの公式**:
```
[みんなが信じていること] なのに、なぜ [現実は逆] なのか？
```

**良いCQの条件**:
1. 視聴者が感じていたけど**言語化できなかった疑問**を代弁している
2. **答えを知るまで離脱できない**
3. **1文15〜25字**で言える

**CQの出し方**:

| パターン | やり方 | 向いている場面 |
|---------|--------|--------------|
| **シングルCQ** | 「なぜ◯◯なのか？」1つだけ | 基本形。迷ったらこれ |
| **トリプルCQ** | 「なぜ？」「誰が得？」「次に何が？」を3連 | 情報量が多いテーマ |
| **二重CQ** | 「なぜ◯◯したのか？＋どうやって？」 | Why＋Howの2軸 |

CQの直後に**約束**（「最後まで見れば分かる」）と**加速宣言**（「では始めましょう」）を入れる。

以下の技法を必要に応じて使う。全部使う必要はない。

| **直接話法** | 「彼はこう言った」で台詞を入れる | ★★☆ |

# ═══ 第六部：感情曲線の設計 ═══

## 感情の種類

| 感情 | いつ使うか |
|------|----------|
| **衝撃** | 冒頭、ゲート直後、どん底 |
| **共感** | 語り手の自己紹介、生活描写 |
| **好奇心** | CQ提示、ナンバリング予告 |
| **理解** | 構造解説、比喩 |
| **恐怖** | エスカレーション、数字の人間化 |
| **怒り** | 不正・搾取の暴露 |
| **悲しみ** | 家族の別れ、喪失 |
| **感動** | 人間の強さ、愛 |
| **危機感** | 日本ブリッジ |
| **余韻** | CTA直前 |

## 2つのルール

**ルール1**: 強い感情（衝撃・恐怖・悲しみ）が**3ブロック以上続いたら息継ぎを入れる**

息継ぎの方法：
- 語り手を交代する
- バランスのデータを入れる（「全てが悪いわけではない」）
- ブラックユーモアを入れる
- 知的解説パートに切り替える

**ルール2**: **1.5〜2分に1回**、語り手またはモード（報道/解説/体験/教訓）を切り替える

# ═══ 第七部：文のリズム ═══

## 文字数の目安

| 場面 | 文字数 |
|------|--------|
| フック（冒頭1〜3文） | **6〜20字** |
| 通常の解説 | **20〜40字** |
| 感情の爆発点 | **3〜10字** |
| CQ | **15〜25字** |
| 教訓・結論 | **25〜40字** |

## リズムの原則

**短→短→短→長→短（爆発）** が基本パターン。

台本全体で**10字以下の短文を5〜10個**配置する。それぞれが**転換点・爆発点・断言**のどれかとして機能する。

# ═══ 第八部：結末の書き方 ═══

## 教訓の設計

教訓は**3項目以内**に圧縮する。箇条書きで視聴者が引用できる形にする。

## 結末の4パターン

| パターン | やり方 |
|---------|--------|
| **リング閉じ＋警告** | 冒頭の言葉を繰り返し、意味を反転させる |
| **チェックリスト** | 崩壊の条件を列挙→「あなたの国を見てください」 |
| **「普通の夢」の崩壊** | 「普通に暮らす夢が高価になりすぎた」 |
| **対称構造** | 2人の語り手が互いに憧れる→構造矛盾を2文で表現 |

## CTAの順番

```
① 問いかけ（「どう感じましたか？」）→ コメントを先に誘発
② 価値の確認（「この動画が役に立ったなら」）
③ 行動指示（「高評価とチャンネル登録を」）
④ 次回への期待（「それではまた次の動画で」）
```

# ═══ 第九部：シリーズ設計 ═══

## 同一国シリーズの構造変化

| 順番 | 構造 | 視点 |
|------|------|------|
| 第1編 | **物語型** | 内側（当事者の体験） |
| 第2編 | **謎解き型** | 外側（地政学・俯瞰） |
| 第3編 | **物語型（サバイバル変形）** | 脱出（逃げた人の視点＋具体的手順） |

## クロスオーバーのルール

- 別の動画の語り手を引用する場合は**1〜2文だけ**
- 初見の視聴者が置いていかれないように前提知識は不要にする

## リサーチの手順

```
Step 1: テーマの基礎データを集める
  → World Bank + IMF + OECD で「数字の地図」を作る

Step 2: CQを見つける
  → データの中から「常識 vs 現実」の矛盾を探す
  → 「みんなが信じていること」と「データが示す現実」のギャップがCQ

Step 3: 語り手を設計する
  → Reddit / Quora / X で現地在住者の声を収集
  → 名前・年齢・職業・家族構成を設定

Step 4: 構造解説の素材を準備する
  → 学術論文・専門家の分析を読む
  → 「日常の言葉で説明できるか」をテスト

Step 5: 数字を人間化する
  → マクロ数字を「1人の1日/1ヶ月」に翻訳するメモを作る

Step 6: 日本ブリッジを設計する
  → 「日本で同じことが起きたらどうなるか」を3パターン考える
```

# ═══ 第十二部：禁止事項 ═══

| 禁止 | 理由 |
|------|------|
| **結論の押しつけ** | 「投資すべきだ」「移民は悪い」は言わない。問いを返す |
| **特定の国・民族への攻撃** | 「問題はシステムであり人ではない」を必ず入れる |
| **データなしの主張** | 全ての主張に数字・出典を |
| **息継ぎのない恐怖の連射** | 3ブロック以上続けない |
| **専門用語の放置** | 出したらその文の中で日常語に翻訳 |
| **冒頭が長い前置き** | 8文以内にCQ |
